・概要
modulesを格納するフォルダ
moduleとは、案件内で頻繁に使われるコンポーネントのようなもの（タイトル、見出し、ボタンみたいなもの）
コンポーネントとの違いは、比較的小さく、出力する場所によって文字列やスタイルが変わるもの

・ルール / 注意点
「m-」以降の「p10」は優先度を示している。数字が小さくなるほど早く読み込まれる
モジュールを一つ作ったら同じ名前のscssファイルを作りモジュールのスタイルは全てそのscssファイルの記述する
例 : 「title-type1」というモジュールを作った時
モジュール m-p30-title-type1.php
scssファイル _p50-title-type1.scss
